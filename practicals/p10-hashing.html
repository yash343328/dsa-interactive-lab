Hash Table

Hash function · Insert · Search · Collision Chaining

// THEORY

A Hash Table maps keys to values using a hash function. Collisions are resolved using chaining (linked list at each bucket).

  • Insert/Search/Delete: O(1) average, O(n) worst (all collide)
  • Load Factor = n/m (n=elements, m=buckets)
  • Collision: two keys map to same index → use chaining or probing
  • Used in: databases, caches, symbol tables, sets
